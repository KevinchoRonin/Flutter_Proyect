Diccionario Técnico: Flutter

Conceptos clave para entender el ecosistema de desarrollo móvil de Google

📱 Desarrollo Mobile

Es la creación de software diseñado para ejecutarse en dispositivos móviles (smartphones y tablets). El ecosistema se divide principalmente en dos enfoques:

  • Nativo: Desarrollar una app exclusiva para cada plataforma (Swift para iOS, Kotlin para Android).
  • Multiplataforma (Cross-platform): Escribir un solo código base que se compila e instala en ambos sistemas operativos.

🚀 ¿Qué es Flutter?

Es un framework y kit de herramientas (UI Toolkit) creado por Google para construir aplicaciones compiladas nativamente para móvil, web y escritorio desde una única base de código.

A diferencia de competidores como React Native, Flutter no usa los componentes visuales del sistema operativo; posee su propio motor de renderizado gráfico (Impeller/Skia) para "dibujar" cada elemento en la pantalla en tiempo real.

🛠️ ¿Qué es un SDK?

Significa Software Development Kit (Kit de Desarrollo de Software). Es el conjunto de herramientas que un programador necesita para construir aplicaciones para una plataforma específica.

El SDK de Flutter incluye el motor de renderizado, herramientas de línea de comandos (CLI), frameworks de pruebas, compiladores y una inmensa biblioteca de componentes de interfaz listos para usar.

🎯 Dart

Es el lenguaje de programación cliente-optimizado en el que está escrito Flutter y el que debes usar para programar tus aplicaciones.

Destaca por combinar dos tipos de compilación: JIT (Just-In-Time) durante el desarrollo para permitir cambios instantáneos, y AOT (Ahead-Of-Time) para producción, que traduce el código directamente a lenguaje de máquina ultra rápido.

🧩 Widgets

En el universo de Flutter, "Todo es un Widget". Un widget es la unidad estructural básica de la interfaz de usuario.

No solo los elementos visibles (un botón o un texto) son widgets; también los elementos estructurales o de diseño (el padding, un alineado o el enrutamiento de pantallas) se declaran y manejan como widgets anidados en forma de árbol.

⚖️ StatelessWidget vs StatefulWidget

Es la distinción fundamental a la hora de estructurar componentes en base a si los datos (el estado) cambian o no en el tiempo:

StatelessWidget (Sin Estado)

Son componentes pasivos e inmutables. Una vez creados con sus propiedades iniciales, no pueden cambiar por sí mismos en la pantalla. Ejemplo: Un texto estático, un icono decorativo o un logo.

StatefulWidget (Con Estado)

Son componentes dinámicos y reactivos. Pueden reconstruirse y actualizar su apariencia visual en tiempo real cuando el usuario interactúa o llegan nuevos datos. Ejemplo: Un contador, una casilla de verificación o un formulario de login.

⚡ Comandos Eficientes (CLI)

Para trabajar de manera profesional con el SDK de Flutter, estos son los comandos imprescindibles en tu terminal de comandos:

flutter doctor Analiza tu entorno local e indica qué componentes o configuraciones faltan por instalar.
flutter create nombre_app Genera automáticamente la estructura de directorios y archivos de un nuevo proyecto limpio.
flutter run Compila, empaqueta y ejecuta la aplicación en un dispositivo físico conectado o emulador.
flutter clean Elimina la carpeta de compilación intermedios y la caché. Es la solución estándar cuando ocurren errores extraños de dependencias.
